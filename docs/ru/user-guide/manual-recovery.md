# Ручное восстановление (расшифровка без Syncrypt)

> Перевод. Источник истины — [English](../../user-guide/manual-recovery.md). При расхождении верна английская версия. Полный скрипт и формат — в английской версии.

Это конкретное доказательство принципа **«данные принадлежат пользователю»**: имея
парольную фразу и документированный формат хранения, ты можешь расшифровать манифест
и каждый файл коротким скриптом — без установки Syncrypt.

> Статус: референсная реализация под crypto-формат версии 1
> ([RFC-0005](../../rfc/RFC-0005-Encryption-Model.md)); поставляется и тестируется с M2.

## Что нужно

- Твоя **парольная фраза**.
- Копия префикса хранилища: `meta/keyfile-params.json` (несекретные соль и параметры
  Argon2id), `manifests/` (зашифрованные манифесты), `objects/` (зашифрованные блобы).

Скачать можно любым S3-клиентом (`aws s3 sync`, `rclone`, веб-интерфейс провайдера).

## Идея

```
MasterKey  = Argon2id(парольная_фраза, соль, параметры)
ContentKey = HKDF-SHA256(MasterKey, "syncrypt/content")
ManifestKey= HKDF-SHA256(MasterKey, "syncrypt/manifest")
```

Формат блоба: `magic("SYNC") | version | alg | nonce(12) | ciphertext | tag(16)`,
шифр AES-256-GCM, заголовок используется как AAD. Расшифровав `manifests/` (самый
свежий по номеру generation) ключом ManifestKey, получаешь соответствие
`путь → objectKey`; каждый объект расшифровывается ключом ContentKey.

**Полный рабочий скрипт на Python** (argon2-cffi + cryptography) — в
[английской версии](../../user-guide/manual-recovery.md#reference-script-python-3).

## Зачем это важно

Если Syncrypt когда-нибудь станет недоступен, заброшен или ты просто ему не доверяешь
— твои данные всё равно полностью восстановимы ~30 строками стандартного кода и
парольной фразой. Никакого lock-in, проприетарного формата и скрытой базы — как
обещано в [RFC-0001](../../rfc/RFC-0001-Vision.md).
