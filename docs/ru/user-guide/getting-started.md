# Быстрый старт

> Перевод. Источник истины — [English](../../user-guide/getting-started.md). При расхождении верна английская версия.

> Статус: движок ещё не реализован (см. [ROADMAP](../../../ROADMAP.md)). Здесь
> описан задуманный процесс v1, чтобы проверить дизайн.

## Что нужно

- Obsidian на каждом устройстве (macOS / Windows / Android).
- S3-совместимый бакет, которым ты управляешь (endpoint, регион, access key,
  secret, имя бакета, опционально префикс).
- Надёжная **парольная фраза**, которую ты не забудешь. **Если потеряешь её —
  данные не восстановить, это заложено в дизайне.**

## Первое устройство

1. Установи плагин Syncrypt в Obsidian и открой настройки.
2. Введи данные хранилища (endpoint, бакет, префикс, ключи).
3. Задай **парольную фразу**. Syncrypt локально выведет из неё ключ шифрования
   (Argon2id) и загрузит только несекретные параметры KDF.
4. Выбери **профиль синхронизации** — что синкать (см.
   [конфигурацию](./configuration.md)). По умолчанию — все заметки и вложения плюс
   безопасное подмножество `.obsidian/`.
5. Нажми **Sync now**. Это первая зашифрованная загрузка и запись первого манифеста.

## Дополнительные устройства

1. Установи плагин, введи **те же** данные хранилища и **ту же** парольную фразу.
2. Нажми **Sync now** → Syncrypt скачает и расшифрует хранилище.

Дальше синхронизация происходит при открытии Obsidian (pull), при закрытии (push),
во время работы (дебаунс) или по кнопке **Sync now**.

## Чтение лога синхронизации

Каждое действие объяснено одной строкой, например:

```
Projects/ATM.md          удалённая версия новее          → скачано
Daily/2026-07-16.md      локальный хеш отличается         → загружено
Old/Deprecated.md        помечен удалённым в манифесте    → удалён локально
Ideas.md                 изменён с обеих сторон           → КОНФЛИКТ (см. копию)
```

Если видишь `Sync stopped. Please pull first.` — другое устройство опубликовало
изменения после твоего последнего pull. Просто сделай pull и повтори push.
