# Устранение неполадок

> Перевод. Источник истины — [English](../../user-guide/troubleshooting.md). При расхождении верна английская версия.

**«Sync stopped. Please pull first.»**
Другое устройство опубликовало более новый манифест после твоего последнего pull.
Сделай **pull** (или Sync now), затем push. Это защитный механизм, а не ошибка.

**Появился файл «conflicted copy».**
Ты (или другое устройство) отредактировал одну заметку с обеих сторон. Syncrypt
сохранил обе версии, а не стал угадывать. Открой обе, сведи нужное в основной файл,
удали копию, синхронизируй снова.

**Не удалось расшифровать / «authentication tag mismatch».**
Либо неверная парольная фраза, либо объект в хранилище повреждён/подменён. Syncrypt
отказывается его применять (fail-closed). Сначала проверь фразу; если верна —
восстанови объект из версии бакета или из бэкапа.

**Похоже на дубликаты заметок между macOS и Windows.**
Почти всегда это несовпадение нормализации Unicode/регистра. Syncrypt нормализует
пути централизованно;
если увидел такое — пришли два точных имени файлов, чтобы воспроизвести.

**Первая загрузка медленная.**
Первый синк шифрует и загружает всё хранилище. Дальше передаются только изменения.

**На Android ничего не синкается в фоне.**
Ожидаемо. Android ограничивает фоновые процессы; синк идёт при открытии/закрытии/во
время работы/вручную.

**Синк захотел удалить/перезаписать много файлов и встал на паузу.**
Это **предохранитель массовых изменений** (Safe Sync). Просмотри список. Если это
ожидаемо (реорганизовал большую папку) — подтверди. Если нет — отмени (ничего не
изменено) и разберись (не тот профиль, не то устройство).

**Файл пропал после того, как синк его удалил.**
Проверь `.obsidian/sync-trash/` на устройстве, где он исчез — Safe Sync хранит
локальную копию перед удалением. Также помогают сохранённые версии и окно tombstone.
