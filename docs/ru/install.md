# Установка и настройка

*Перевод; источник истины — [английская версия](../install.md).*

Syncrypt сейчас в бете и ставится через
[BRAT](https://github.com/TfTHacker/obsidian42-brat) — стандартный способ
пробовать пре-релизные плагины Obsidian. Шаги одинаковы на Windows, macOS и
Android.

## 1. Установи BRAT

Obsidian → **Settings → Community plugins** → Browse → найди **«BRAT»** →
Install → Enable.

## 2. Добавь Syncrypt через BRAT

1. **Settings → BRAT → Add beta plugin.**
2. Введи репозиторий: `Dyms/syncrypt`.
3. Подтверди — BRAT скачает свежий релиз.
4. **Settings → Community plugins** → включи **Syncrypt**.

Обновления: BRAT сам следит за релизами; «Check for updates» подтянет свежую
бету.

## 3. Подготовь хранилище (один раз, с любого устройства)

Нужен либо **S3-совместимый бакет**, либо **WebDAV-папка**.

**S3 (AWS, MinIO, Cloudflare R2, S3 хостинг-провайдера …):**

- Создай бакет, выделенный под это хранилище.
- Создай ключи с **минимальными правами**: доступ только к этому бакету, без
  админских прав. Syncrypt никогда не создаёт и не удаляет бакеты.
- Настоятельно рекомендую включить **versioning бакета** — это страховка от
  порчи шифротекста кем/чем угодно с правом записи.

**WebDAV (Nextcloud, Apache …):** создай отдельную папку и, если сервер умеет
(Nextcloud умеет), отдельный **app password**.

## 4. Настрой Syncrypt

**Settings → Syncrypt:**

- **S3**: endpoint, регион, бакет, необязательный префикс (подпапка в
  бакете), access key ID + secret. *Path-style addressing* оставь включённым
  для MinIO/R2/self-hosted; некоторым конфигурациям AWS нужен выключенный.
- Важно: креденшелы хранятся в файле настроек плагина на этом устройстве
  (заметки защищает passphrase, которая *не* хранится). Поэтому и нужны
  least-privilege ключи.
- Сеть ходит через нативные запросы Obsidian — настраивать CORS на бакете или
  сервере не нужно.

## 5. Первая синхронизация

1. Выполни команду **«Syncrypt: Unlock»** (или кнопку Unlock в настройках).
2. Введи passphrase. На самом первом устройстве это *создаёт* шифрование
   хранилища; **выбери сильную passphrase и сохрани её в менеджере паролей —
   восстановить её невозможно.**
3. Если среди устройств есть телефон — оставь профиль шифрования
   **cross-device** (по умолчанию).
4. Выполни **«Syncrypt: Sync now»**. Первая синхронизация зашифрует и зальёт
   всё хранилище; дальше передаются только изменения.

## 6. Каждое следующее устройство — простой путь

На уже настроенном устройстве выполни **«Syncrypt: Share connection»**.
Команда спросит passphrase и выдаст **connection ticket** — одну длинную
строку со всеми настройками хранилища и ключами, зашифрованную под твою
passphrase. Без passphrase тикет бесполезен.

На новом устройстве: поставь Syncrypt (шаги 1–2), выполни **«Syncrypt: Add
this device from a ticket»**, вставь тикет, введи ту же passphrase — готово.
Всё заполнится и подключится само. После этого **удали сообщение или заметку,
через которую переносил тикет**: он зашифрован, но относись к нему как к
секрету.

Не хочешь класть ключи в тикет вовсе? Сними галку «Include storage
credentials» при экспорте — тикет понесёт только конфигурацию, а ключи введёшь
на новом устройстве руками.

(Тикеты можно вообще пропустить и ввести те же настройки + passphrase
вручную, как на первом устройстве.)

На Android дефолты бережны к батарее и трафику: авто-синхронизация ждёт
Wi-Fi (ручная работает всегда), выполняется только на переднем плане, а
проверка «без изменений» стоит считанные килобайты.

## 7. Как понять, что всё синхронизировано?

Смотри в **статус-бар** (низ окна): `synced ✓` значит ровно это — все
локальные изменения загружены и ты на последнем опубликованном состоянии.
`pending` — что-то ещё ждёт синхронизации (тултип скажет, что именно). Также
бывают `syncing…`, `offline` и `conflict (n)`. Зелёная галка никогда не
появляется раньше правды. Клик по статусу запускает Sync now; та же
информация — вверху вкладки настроек.

## Мигрируешь с другого инструмента синхронизации?

Сначала выключи его — два синка на одном хранилище передерутся. Syncrypt
предупредит, если что-то обнаружит. Полный гайд:
[миграция с LiveSync](./user-guide/migration-from-livesync.md).

## Если что-то не работает

[Решение проблем](./user-guide/troubleshooting.md): ошибки «Unauthorized»
из-за сбитых часов, conflicted copies, статус «waiting for Wi-Fi» и другое.
**Лог синхронизации** («Syncrypt: Show sync log») объясняет каждое действие с
причиной — начни с него.
